Tasks to complete before you generate the Record Distribution to WebSphere MQ: one-way pattern

Before you generate a pattern instance, you must complete the following tasks:

  • Confirm that this pattern meets the business requirements and can be supported by both the system that creates the files and the systems to which the files are sent.
  • Identify the purpose of the pattern application and give the pattern instance an appropriate name.
  • If logging is required, decide which queue manager and queue will receive the logging messages.
  • Decide how records are detected and if validation is required. If necessary, prepare a message model to define how records are parsed.
  • Determine whether the files that are to be processed are local or remote. If remote files are to be processed, identify the name of the FtpServer configurable service. For more information, search for "FtpServer configurable service" in the IBM Integration Bus documentation.
  • Decide if multiple possible destinations exist. If multiple possible destinations exist, select the routing approach to use.
    • If you have a limited number of routes where the destinations do not change frequently, set the Routing parameter to Specify routes.
    • If the set of target routes is not fully determined or might change frequently, set the Routing parameter to Lookup routes. If this value is selected, the name of the data source must be known.
